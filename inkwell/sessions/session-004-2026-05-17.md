# Session 004 — 2026-05-17

## What was done

Three improvements to the AI features, all committed and pushed to prod.

### Fast/Smart model toggle

Added a segmented Fast/Smart toggle to both the AI edit panel (note.js) and the AI chat page (chat.html/chat.js).

- Fast = `claude-haiku-4-5-20251001`
- Smart = `claude-sonnet-4-6`
- Choice persisted in localStorage as `inkwell-ai-mode`, shared between both pages
- Toggle renders as an inline segmented control with inverted fill on the active option

### Improved AI edit system prompt

Researched prompt engineering best practices and rewrote the AI edit system prompt to make surgical edits when the instruction is narrow, rather than always regenerating the full note.

The new prompt asks the model to classify the instruction as NARROW or BROAD inside `<scope>` tags before producing output, then apply different rules per classification:
- NARROW: copy every unchanged sentence word for word, only touch what the instruction requires, default to leaving content unchanged if scope is uncertain
- BROAD: rewrite freely, preserve voice and core meaning

The `<scope>` tag is stripped from the response before displaying the preview, so the user never sees it.

### Generate note sends pending input first

When the user has text typed in the chat input and clicks Generate note, that text is now sent as a user message first (appended to the chat and added to history), then generation runs with the updated history. If the input is empty, behaviour is unchanged.

## Decisions made

**Scope classification via chain-of-thought.** Making the model commit to NARROW or BROAD before generating primes its context for the correct output strategy. This is more reliable than a single instruction to "preserve unless asked to change" because it forces an explicit reasoning step before output begins.

**Shared localStorage key for model toggle.** Both AI edit and chat use the same `inkwell-ai-mode` key so the user's preference carries across both surfaces without separate settings.

**Pending input on Generate note.** This gives the user a natural way to add a final instruction or context (e.g. "focus on the technical decisions") alongside triggering note generation, without needing to send and then immediately click generate.
