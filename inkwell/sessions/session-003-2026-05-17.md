### Bug 3: db.js DataError on IDBKeyRange.only(null)

**Error:** `DataError: Failed to execute 'only' on 'IDBKeyRange': The parameter is not a valid key.`

**Cause:** A leftover variable `const key = parentId === null ? IDBKeyRange.only(null) : ...` from an earlier draft. `IDBKeyRange.only(null)` throws because `null` is not a valid IndexedDB key. The variable was never used — the correct `getAll()` + filter approach was already on the next line.

**Fix:** Removed the dead line entirely. Commit `f35ed01`.
