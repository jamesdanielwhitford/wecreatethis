# Session 005 - 2026-07-14

## Summary

Full rebuild of the `blog/` app from a hardcoded single-post design into a folder-driven, scroll-through blog with a hand-authored homepage. All work on branch `blog-work` (off `dev`), committed in three stages.

## What was done

**1. Folder-based content structure**
- Old: `blog/posts/*.md` + a hardcoded `POSTS` array in `app.js`.
- New: `blog/content/{section}/{post-slug}/index.md`, each post with YAML frontmatter (`title`, `date`, `author`, `description`, optional `order`).
- Added `blog/build.js` (plain Node, no deps) that walks `content/` and writes `blog/content-manifest.json`, which the site fetches at runtime. Run manually before deploy (`node blog/build.js`), same "no build step, just remember to run a script" philosophy as the rest of the site.
- Migrated the existing post ("How to write (sick) loops in Claude Code") into `content/dev-tools/how-to-write-sick-loops-in-claude-code/index.md`.
- Cleaned up orphaned cruft found along the way: a stale `blog/images/` folder and a `sw.js` reference to a post (`claude-is-skeptical-about-openclaw.md`) that had been deleted in an earlier commit but left orphaned assets behind.

**2. Scroll-through post stack + clean URLs**
- Visiting a section now renders every post in that section as one continuous scroll (not paginated) — scrolling past the end of one post flows straight into the next, in both directions.
- URL scheme: `/blog/{section}` for a section, `/blog/{section}#{post-slug}` to land on a specific post within it. The post identifier is a URL **fragment**, not a path segment or query string, so the browser handles the initial scroll position natively and Cloudflare Pages needs zero server-side routing for individual posts.
- `post.html` was merged into `section.html` (one page renders the whole stack).
- Added one `_redirects` rule for the clean `/blog/{section}` path. Cloudflare's placeholder rule matching doesn't check whether a real file already exists at that path first, so it was initially swallowing `/blog/app.js`, `/blog/manifest.json`, etc. and serving them as HTML. Fixed by listing each real static file in `blog/` as an explicit pass-through rule *above* the wildcard (order matters, first match wins) — same "add it to a list" discipline the project already uses for the SW `ASSETS` array.
- Added a `blog/content/demo/` section (3 short posts: post-one/two/three) purely to exercise and demo the scroll-through behavior without touching real content.

**3. Hand-authored homepage**
- Originally the homepage auto-listed sections from the manifest. Changed direction mid-session: homepage is now driven by `blog/content/home.md`, rendered directly like a post at `/blog/`.
- This means visual grouping/headings ("Dev Tools", "Demo", and future ones like "Game Development") and which sections show up at all, in what order, are edited by hand in one markdown file — not derived from folder structure. Simpler than the folder-based "group" level that was briefly considered and discarded (see below).
- Added unordered-list support to the minimal markdown renderer (`app.js`) since the homepage needed `- [text](url)` link lists; renderer still only covers headers/bold/italic/links/lists/paragraphs, no tables/blockquotes/code yet (explicitly deferred, per original ask).

## Notable dead end (discarded, for context)

Briefly explored adding a folder-based "group" level above sections (`content/{group}/{section}/{post}/index.md}`, e.g. a "Game Development" folder containing multiple section folders) so the homepage could visually separate posts by group. Abandoned in favor of the hand-authored `home.md` approach once the user realized editing one file directly was simpler than encoding grouping into folder structure. No group-folder code was ever written to disk, safe to ignore if you see it mentioned upstream in conversation history.

## Local preview

Static servers (Python `http.server`, `npx serve`) don't understand Cloudflare's `_redirects` file, so clean URLs like `/blog/dev-tools` 404 under them — not a bug, just a local tooling limitation. Correct local preview is Cloudflare's own emulator:

```
npx wrangler pages dev . --port 8080
```

Then visit `http://localhost:8080/blog/`. This serves the whole repo (all apps), not just the blog, since `_redirects` is repo-wide.
