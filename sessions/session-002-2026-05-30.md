# Session 002 — 2026-05-30

## What was done

### Voice Notes — remove live streaming, fix notes not loading, add mic-drop detection

**Bug fixed: notes never loaded from IndexedDB**
The `getNotes()` function queried a `created_at` index but all notes are stored with a camelCase `createdAt` property. The index never matched anything, so the list was always empty after reload. Fixed by bumping to `voice-notes-v3` DB and recreating the index as `createdAt`.

**Live streaming removed**
Removed the 10-second chunked transcription interval (`chunkTimer`, cumulative blob diff, live transcript box UI and CSS). Replaced with a simpler flow: note saved to IndexedDB immediately as `transcribing`, single Mistral call made after recording stops, card updates in-place when result arrives.

**Mic-drop detection added**
Two new event listeners on `startRecording`:
- `mediaRecorder.onerror` — catches recorder-level failures
- `track.addEventListener('ended', ...)` on each stream track — catches OS/hardware mic disconnect

Either fires `micDropped()`, which: stops the pulse animation immediately, shows "Mic disconnected" status + toast, resets to "Tap to record" after 3 seconds. Previously the button would keep pulsing after the mic was gone.

**SW bumped to v13.**

**Pushed to prod** — committed and pushed to `main`, Cloudflare Pages deploying.

## Decisions made

- **Single transcription call on stop** — simpler, cheaper, no API cost during recording, avoids the cumulative-blob complexity that caused the 400 bug last session.
- **DB version bump to v3** — required to recreate the index with the correct key path. Notes recorded before this session on v2 won't carry over (there were none that successfully loaded anyway due to the bug).
