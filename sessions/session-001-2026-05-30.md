## Decisions made

- **Auth method:** username + password stored as `btoa(user:pass)` Bearer token. Credentials cached in `localStorage` (if "stay signed in" checked), session token in `sessionStorage` (clears on tab close). Chosen over single token for better UX ‚Äî readable credentials, standard login form.
- **10s chunks, not realtime WebSocket:** `voxtral-mini-realtime-latest` is available on the API key but chunked batch was chosen ‚Äî simpler, no WebSocket management, same live feel at 10s intervals.
- **No audio stored remotely:** D1 only stores metadata + transcript text. Audio blobs stay on device only. Simpler, cheaper, avoids R2 setup for this use case.
- **Sync strategy:** Remote always wins on upsert. Admin device is considered authoritative for remote state.
- **IndexedDB split:** Metadata in `voice-notes-v2`, audio blobs in `voice-notes-audio` (separate DB). Keeps note metadata fast to query without loading blob data.

## Bug fix (same session)

Live chunked transcription returned 400 from Mistral. Root cause: sending only the last 10s of raw `MediaRecorder` chunks produces an invalid webm file ‚Äî webm requires the initial header/cluster data to be decodable. Fixed by sending the **full cumulative blob** (all chunks from recording start) every 10 seconds, then diffing the returned text against the previous full transcript to extract only the new portion. Final call on stop does the same. SW bumped to v12.
