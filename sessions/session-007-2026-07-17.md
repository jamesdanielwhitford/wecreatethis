## Decisions

- **Final canonical strategy (Act 2)**: stale-while-revalidate with update notification. Cached responses serve instantly; a background fetch revalidates; if a shell asset changed (header-only `responsesDiffer`: ETag, Content-Length, Last-Modified), the SW posts `{type: 'sw-updated'}` and the shared root `/sw-toast.js` shows a refresh pill. Reload is always user-initiated so games in progress are never yanked. Root `sw.js` (v25) is the exemplar all apps follow.
- **`cleanResponse()` is mandatory on every cache.put**: Cloudflare's `_redirects` self-rewrites mark responses `redirected: true`, and serving those to a navigation fails with ERR_FAILED (rediscovered live via Playwright; tarot's old notes mention the same bug).
- **Superseded (Act 1)**: network-first with `fetchWithTimeout` (3000ms). Committed as `01a5150`, replaced the same day; kept in history.
- **Canonical URLs**: extensionless everywhere; internal `.html` links being fixed in git-notes, longform, pokemon, starrynight.
- **Version bumps now near-obsolete**: with network-first, deploys reach users immediately; CACHE_NAME bumps only purge deleted files. All SWs bumped once for this migration.
- **Preserved exceptions**: blog SW untouched (already network-first for content, session-006 design); perfectday tile passthrough + CDN branch; symbolicritual media cache (R2/LRU); tarot CACHE_DECK/REMOVE_DECK message handler; birdle + chessle cross-origin caching (no timeout on cross-origin).
- **Local testing**: `npx wrangler pages dev .` is the standard; `dev/serve.json` cleanUrls hack obsolete (deletion of `dev/` blocked by permissions, left for James).
