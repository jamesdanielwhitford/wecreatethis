# Session 006 - 2026-07-17

## Summary

Blog upgraded to the "just add folders" model: recursive nested sections, unified breadcrumb layout, routing that never needs per-post changes, CI manifest regeneration, a reading-order toggle, and code/blockquote rendering with a tiny built-in highlighter. All work on `dev`, verified end-to-end with Playwright against `wrangler pages dev`.

## What was done

**1. Recursive content model (`build.js`)**
- Sections can now nest to any depth: any folder with an `index.md` is a post, its parent path (relative to `content/`) is the section, e.g. `content/game-dev/godot/my-post/index.md` renders at `/blog/game-dev/godot`.
- Manifest sections keyed by `path` (was `slug`); a post folder may itself contain further post folders.
- Posts directly under `content/` (no section) are skipped with a warning.

**2. Unified layout**
- New shared `blog/style.css`; inline styles removed from both pages.
- Both pages render the same breadcrumb header via JS: `wecreatethis.com / blog / section / subsection`, every segment a link except the current one.
- Section pages list nested subsections above the posts, so intermediate paths (a "group" folder with no direct posts) work as index pages instead of 404ing.
- Post bodies and the home page share one `.md-content` style block.

**3. Routing with zero per-content maintenance (`_redirects`)**
- Replaced the single-segment `/blog/:section` rule with: `/blog/ /blog/ 200` (home), `/blog/content/* /blog/content/:splat 200` (content pass-through), explicit pass-throughs for static files, then `/blog/* /blog/section 200` for any section depth.
- Gotcha discovered: Cloudflare/wrangler silently ignores any rule whose target is `.../index.html` ("infinite loop detected", its pretty-URL logic strips `.html`/`index`), which is why `/blog/ /blog/index.html 200` never applied and the wildcard swallowed the homepage. Self-rewrite `/blog/ /blog/ 200` is the working pattern.

**4. CI manifest regeneration**
- New `.github/workflows/blog-manifest.yml`: on pushes to `dev`/`main` touching `blog/content/**` or `build.js`, runs `node blog/build.js` and commits the manifest back if changed. Manual `node blog/build.js` still works and keeps CI a no-op.

**5. Reading-order toggle**
- Section pages with 2+ posts show a "Reading: newest first / oldest first" toggle that reverses the stack (label derived from actual post dates; falls back to "Reverse order" for `order`-driven sections). Not persisted.

**6. Markdown renderer extensions (`app.js`)**
- Fenced code blocks, inline code, and blockquotes added (tables still deferred).
- Blockquotes are rendered recursively (prefix-stripped, re-fed through the renderer), which the existing dev-tools post needed: it contains a code fence inside a blockquote.
- Extraction uses NUL-delimited placeholders so code content can never collide with prose or be mangled by other transforms.
- Tiny language-agnostic highlighter (~25 lines): comments (`//`, `/* */`, line-start `#`), strings, numbers, shared keyword set. Muted colors in `style.css` (`.tok-c/s/n/k`). No packages.

**7. Service worker rework (`sw.js`, v6 → v7)**
- `ASSETS` is now shell-only; post/content URLs are derived from `content-manifest.json` at install time. Adding a post no longer requires touching `sw.js` at all.
- Content + manifest are network-first with cache fallback, so new posts appear to installed clients without a version bump. Shell stays cache-first.
- Offline navigation to a never-visited section falls back to the cached `section.html` shell (verified).
- **Cache-poisoning bug found and fixed:** fetching `/blog/index.html` at install returned `section.html` (wildcard redirect rule), and its cached body landed under the normalized key `/blog/`, replacing the homepage. Fix: `ASSETS` lists canonical extensionless URLs only; never fetch `.html` URLs from the blog SW.
