# Session 001 — 2026-05-20

## What was done

Built the Chess Timer app from scratch inside `/chesstimer/` as a new PWA in the wecreatethis multi-project repo.

### Setup screen
- Hours/minutes/seconds drum-roll picker (drag up/down to change value) with inline unit labels ("5 min", "0 sec") and a fade gradient top/bottom
- "Chess Timers" heading left-aligned
- "Previous Timers" list with tap-to-reuse and swipe-delete (red X button)
- Flat orange Start button (no 3D shadow)

### Ready screen
- Two full-height green half-panels, top one rotated 180° so each player faces their own "press to start"
- The player who taps their half becomes the first active player
- Red cancel button in center bar between the halves

### Active timer screen
- Two rounded-rect panels (orange = active, grey = inactive), each filling half the screen with 8px margin inset
- Center bar between panels holds cancel (red, left) and pause/resume (yellow/green, right)
- Flat buttons, no 3D shadow
- Top panel rotated 180° so each player reads their own time right-side up
- Tapping your own panel switches the turn to the other player

### Pause behavior
- Pause button toggles to a green play button
- Tapping either player panel also resumes
- Game auto-pauses when exit modal is shown

### Exit modal
- Two half-screen modals stacked (top rotated 180°) so both players can read it
- "End Game" (red) and "Keep Playing" (grey) — either player can dismiss

### Styling
- iOS Calculator colour palette (style guide in `STYLE_GUIDE.md`)
- All buttons flat, no box-shadow
- Cancel X icon uses `--red-dark`, pause icon uses `--yellow-dark`
- `100dvh` for full viewport fill

### PWA
- `manifest.json`, `sw.js` (cache-first, offline support), icons generated
- Added to wecreatethis homepage `index.html`

## Decisions made

- Used a flex `center-bar` child between the two panels rather than absolutely positioned buttons, so action buttons never overlap the rounded corners of the player panels
- Drum picker built with touch/mouse drag events rather than native `<input type="number">` for better mobile UX
- Exit modal split into two halves (not rotated 90°) for readability by both players
- `position: fixed; height: 100dvh` on screens to fill full mobile viewport
