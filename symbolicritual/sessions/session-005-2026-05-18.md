# Session 005 — 2026-05-18

## What was done

- Diagnosed and fixed items not appearing on prod: `visibility: hidden` CSS was set on `.item` but Safari iOS was not picking up the inline `visibility: visible` override. Switched to `opacity: 0` / `opacity: 1` with a `0.15s` fade transition instead.
- Fixed the root cause of 401 Unauthorized errors on admin actions: `api.js` had its own separate `getToken()` function still reading from `sessionStorage`, while `admin.js` had been updated to `localStorage`. All API calls were sending an empty token. Fixed `api.js` to read from `localStorage`.
- Deferred initial padding measurement to `requestAnimationFrame` so layout dimensions are available before `applyItemPadding` runs.
- Added `load` and `loadedmetadata` event listeners on media elements to recalculate padding after images/videos finish loading.
- Items now hidden (`opacity: 0`) until padding is correctly calculated, then fade in — no layout jump visible to the user.
- Bumped SW through v11, v12, v13, v14 across fixes this session.
- Added `?mock` URL mode for local dev: loads 5 hardcoded test items from `test-assets/` without touching IndexedDB or the API.

## Decisions made

- `opacity` over `visibility` for the hide-until-ready pattern — inline style overrides work more reliably across browsers.
- `getToken()` kept as a local function in both `admin.js` and `api.js` rather than sharing — avoids a module dependency for a one-liner, but both must be kept in sync.
