# Session 008 — 2026-05-18

## What was done

Investigated mobile scroll stutter on the symbolic ritual feed. Identified two causes and fixed both.

**Fix 1: `dvh` units replaced with `svh`**

`min-height: 100dvh` and `margin-bottom: 12.5dvh` on `.item` caused continuous reflow on iOS Safari. The dynamic viewport unit (`dvh`) recomputes whenever the browser chrome (address bar) shows or hides during scroll, triggering layout recalculation on every `.item` element mid-scroll. Switched to `svh` (small viewport height), which is a fixed value tied to the smallest viewport state and never recomputes during scroll.

**Fix 2: Removed `content-visibility: auto`**

`content-visibility: auto` was hiding off-screen items and painting them as they entered the viewport, causing a visible hide/unhide jank on scroll. With images already using `loading="lazy"` and explicit `width`/`height` attributes, native lazy loading handles deferred rendering without the layout recalculation cost. Removed `content-visibility: auto` and `contain-intrinsic-size` from `.item`.

SW bumped to v24, pushed to prod.

## Decisions

- `svh` rather than `vh`: `vh` uses the largest viewport (same issue as `dvh` on some browsers). `svh` is the safest fixed value for scroll-stable layouts.
- Posts will be sized slightly shorter (as if the address bar is always visible) rather than sometimes taller. Acceptable trade-off for smooth scrolling.
