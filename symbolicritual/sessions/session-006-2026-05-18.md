# Session 006 — 2026-05-18

## What was done

- Replaced JS-measured item padding with pure CSS: `.item { min-height: 100dvh; display: flex; justify-content: center; padding-block: 32px; content-visibility: auto; contain-intrinsic-size: auto 100dvh }`. Math is identical to the old `applyItemPadding` but the browser does it at first paint — no measurement pass, no stutter on image load.
- Removed everything supporting the old measurement pipeline from `app.js`: `applyItemPadding`, `itemResizeObserver`, the `load`/`loadedmetadata`/rAF triggers, the window `resize` listener, and the `opacity: 0 → 1` fade-to-show pattern.
- `app.js` URL tracking: replaced `visibilityMap` + per-callback `Map` recomputation with a single IntersectionObserver using `rootMargin: '-50% 0px -50% 0px'`. The item crossing the viewport-centre band is "current". `history.replaceState` calls are rAF-throttled.
- Added a second IntersectionObserver (`rootMargin: 150%`) that upgrades `fetchPriority` from `low` → `high` as an item approaches the viewport. First item gets `loading="eager"`, `decoding="sync"`, `fetchPriority="high"`; everything else starts `lazy`/`async`/`low`.
- `sw.js` rewrite: previously cross-origin R2 media was never cached (the SW returned early for cross-origin requests). Added a dedicated `symbolic-ritual-media-v1` cache with cache-first behaviour, LRU trim at ~200 MB using `Content-Length`, Range-request passthrough for `<video>`, and a forced-CORS re-fetch so opaque responses don't pollute the cache. App-shell cache bumped to `symbolic-ritual-v15`.
- `db.js`: added `putItems(items)` that writes N items in one transaction. `syncFromApi` now does one IndexedDB round trip instead of N.
- `syncFromApi` rewritten to bucket items into newer/older/middle before insertion. No more per-item `querySelectorAll` in the common case.
- Bumped SW to v15, committed on dev, merged to main, pushed to prod.

## Decisions made

- CSS over JS for layout — `min-height: 100dvh; justify-content: center` produces the exact same visual as `pad = max(32, (vh - contentHeight)/2)` did, but with zero JS work and stable from first paint. The user's "quarter-screen gap" property is preserved because flex centering naturally leaves whitespace equal to half the spare space on each side of the figure.
- Kept `max-height: 100dvh` on `.item-media` (rather than `calc(100dvh - 64px)`) so fullscreen-bleed images still touch the viewport edge. The 32px padding above is intentional — same as the previous `MIN_GAP`.
- `content-visibility: auto` on every item for free off-screen virtualisation. Safari < 18 ignores it harmlessly; modern browsers skip rendering off-screen items entirely.
- Force CORS in the SW media fetch (`new Request(url, { mode: 'cors', credentials: 'omit' })`). R2 sends `ACAO: *` so the upgrade is safe, and the resulting response has readable `Content-Length` for the LRU trim. Without this, `<img>`'s default no-cors mode would yield opaque responses with no introspectable size.
- Range requests for `<video>` bypass the cache entirely. The Cache API doesn't honour byte ranges and returning a full body to a range request can confuse the player.
