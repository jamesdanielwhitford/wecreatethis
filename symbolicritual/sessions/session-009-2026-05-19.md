# Session 009 — 2026-05-19

## What was done

Three changes shipped to prod across SW v25, v26, v27.

**Fix: local-first cache too aggressive — stale content never updated**

The app rendered from IndexedDB on every load and only called `syncFromApi()` when the feed was completely empty. Edited items (caption, coords, etc.) never updated on screen because `syncFromApi` skipped slugs already in `renderedSlugs`.

Fixed by:
- Stamping `article.__itemData = item` on each rendered article so change-detection has something to compare against.
- Adding `itemChanged()` which checks a set of watched fields (`caption`, `alt`, `lat`, `lng`, `captured_at`, `media_url`, `media_type`, `lang`, `width`, `height`).
- In `syncFromApi`, already-rendered items are now re-rendered in place via `replaceChild` if any watched field differs from the live API response.
- `init()` now always awaits `syncFromApi()` at the end, regardless of whether the cache had data. Flow: render from cache first (instant), then sync from API (patches edits, inserts new items).

**Post gap reduced from 12.5svh to 10px**

The previous `margin-bottom: 12.5svh` (~90px on a phone) was too large. Reduced to a flat 10px.

**Lightbox: tap image to expand full screen**

Tapping any image opens a full-screen dark overlay with the image filling the viewport. Tapping the overlay (or pressing Escape on desktop) closes it. Implemented with a `#lightbox` div, CSS `position: fixed; inset: 0`, and a click listener in `app.js`. Videos are unaffected (they have native controls). Cursor changes to zoom-in/zoom-out to hint at the interaction.

## Decisions

- Re-render on change rather than patch individual fields: simpler, no risk of partial DOM state, cost is negligible for ~20 items.
- Always await sync on init rather than fire-and-forget: slight delay before "nothing here" message is acceptable; ensures the empty state is accurate.
- 10px gap as a flat px value rather than viewport-relative: the gap is decorative, not layout-critical, so a fixed value is fine.
- Lightbox reuses the same `<img>` element across opens to avoid repeated DOM allocation.
